Разбор команды df -h | grep '/$' | awk '{print $4}'

Давайте разберемся, что происходит в приведенной цепочке команд и почему мы получаем именно такой результат.

Исходная команда:

df -h | grep '/$' | awk '{print $4}'

Разбираем каждую часть отдельно:

Шаг 1: df -h

Эта команда выводит информацию обо всех файловых системах вашего компьютера, показывая общий объём, использованное пространство, доступное пространство и процент занятости в удобочитаемом формате («-h»).

Пример вывода:
Filesystem                         Size  Used Avail Use% Mounted on
udev                               938M     0  938M   0% /dev
tmpfs                              198M  1,1M  197M   1% /run
/dev/mapper/ubuntu--vg-ubuntu--lv   12G  4,9G  5,8G  46% /
tmpfs                              986M     0  986M   0% /dev/shm
...

Шаг 2: grep '/$'

Команда grep фильтрует строки, содержащие символ "/" в конце строки (/$). Здесь мы ищем строку, соответствующую корневой файловой системе "/".

Фильтрация строк с "/" :
/dev/mapper/ubuntu--vg-ubuntu--lv   12G  4,9G  5,8G  46% /

Шаг 3: awk '{print $4}'

Команда awk обрабатывает каждый столбец в строке. В данном случае "$4" выбирает четвертый столбец из найденной строки. Четвертый столбец соответствует полю «Доступно».

Выбор четвертого столбца:
5,8G

Итак, весь процесс выглядит следующим образом:

  1. Запускаем команду df -h, чтобы увидеть информацию о файловых системах.
  2. Применяем фильтр grep '/$', чтобы выбрать строку, относящуюся к корневому каталогу («/»).
  3. Используем awk, чтобы извлечь четвертый столбец из полученной строки, представляющий доступное пространство.

Итоговый результат:

5,8G

Это значение отражает доступное пространство на корневой файловой системе («/»).
